# sock_predict

単一HTMLで動作する、株式分析ツールです。  
1つの画面で次の3モードを切り替えて使えます。

- テクニカル分析
- ファンダメンタル分析
- 行動ファイナンス

実体ファイルは [index.html](index.html) です。

## 主な機能

### 1. テクニカル分析
- コンフルエンススコアの算出（BUY / WAIT / SELL）
- 指標群
  - SMA/EMA
  - MACD
  - RSI
  - ストキャスティクス
  - ボリンジャーバンド
  - ATR
  - OBV
- パターン検出
  - ダブルトップ/ボトム
  - ヘッドアンドショルダー
  - トライアングル
  - フラッグ
- サポート/レジスタンス
- フィボナッチ・リトレースメント
- Lightweight Charts による可視化

### 2. ファンダメンタル分析
- 日本株（4桁コード）で J-Quants 連携に対応
- バリュエーション表示
  - PER
  - フォワードPER
  - PBR
  - EV/EBITDA
  - PSR
- 収益性・効率性
  - ROE/ROA
  - 利益率
  - EPS/BPS
- 財務健全性
  - 自己資本比率
  - 純資産
  - 総資産
  - D/E など（取得可能時）
- CF・成長性
  - 営業CF
  - FCF（営業CF + 投資CF）
  - 売上/利益成長率
  - 配当利回り・配当性向
- グレアム理論値と安全域（Margin of Safety）

### 3. 行動ファイナンス
- バイアスリスク判定（LOW / CAUTION / HIGH）
- 判定根拠の表示
- エントリー前チェックリスト
- ATRベースのポジションサイジング（参考）

## 動作環境

- Windows / macOS / Linux
- モダンブラウザ（Edge / Chrome 推奨）
- ローカルHTTPサーバー

## 使い方

### 1. ローカルサーバー起動

作業ディレクトリで実行:

    python -m http.server 8099 --bind 127.0.0.1

次をブラウザで開く:

- http://127.0.0.1:8099/index.html

注意:

- file:///C:/.../index.html で直接開くと CORS 制約で取得失敗しやすくなります。

### 2. シンボル入力

- 日本株: 7203（4桁）
- 米国株: AAPL
- 指数: N225

### 3. J-Quants APIキー設定（日本株の財務指標用）

画面右上の API設定 から登録します。

1. J-Quants に無料登録
2. ダッシュボードで APIキー を発行
3. 画面モーダルに貼り付けて保存

保存先はブラウザの localStorage です。

## データソース

- 価格・時系列: Yahoo Finance v8 chart（フォールバックあり）
- 財務サマリー: J-Quants /v2/fins/summary（日本株）

## 想定する時間軸

### テクニカル分析
- 想定: 日足ベースのスイング（数日〜数週間、長めで数か月）
- 理由: 日足データ（interval=1d）と1年レンジ（range=1y）で判定しているため
- 補足: 1M/3M/6M/1Y は表示範囲の切替であり、判定ロジック自体は日足前提

### ファンダメンタル分析
- 想定: 中長期判断（数か月〜1年以上）
- 理由: J-Quantsの開示ベース（四半期/通期）を使うため、短期売買シグナルには不向き
- 主用途: 銘柄選別、保有継続判断、割安/割高の大枠判断

### 使い分けの目安
- ファンダメンタル: 何を買うか（銘柄選定）
- テクニカル: いつ入るか/いつ出るか（タイミング）

## 指標の制約（重要）

- EV/EBITDA
  - /v2/fins/summary では銘柄・期間・開示様式によって空欄になりやすく、N/A になる場合があります。
- PSR
  - 売上と時価総額が揃えば算出可能です。
  - ただし、株式数や売上の欠落時は N/A になります。
- IFRS/USGAAP 等では一部項目が空欄になることがあります（J-Quants仕様の留意点）。

## よくあるトラブル

### N/A が多い
- APIキーが設定されているか確認
- HTTPで開いているか確認（file:// は避ける）
- 銘柄コード形式を確認（日本株は4桁）

### python サーバー起動が Exit Code 1
- 既に同ポートで待受している可能性があります。
- その場合は既存プロセスを使って http://127.0.0.1:8099/index.html を開いて問題ありません。

### すぐ確認したい
- 強制再読み込み（Ctrl+F5）後に再実行
